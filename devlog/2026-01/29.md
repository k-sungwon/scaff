## 🎯 핵심 개념 (면접 대비)

### 1. **Next.js 15 Dynamic Routes** ⭐⭐⭐

```tsx
// app/problems/[subject]/[year]/[examType]/[number]/page.tsx
type Props = {
  params: Promise<{
    // Next.js 15에서는 Promise
    subject: string;
    year: string;
    examType: string;
    number: string;
  }>;
};

export default async function ProblemPage({ params }: Props) {
  const { subject, year, examType, number } = await params;
  // ...
}
```

**면접 답변**: "Next.js 15부터 params가 Promise로 변경되어서 await으로 값을 받아야 합니다. 동적 라우트의 파라미터를 TypeScript로 명확히 타입 정의했습니다."

### 2. **서버 컴포넌트 vs 클라이언트 컴포넌트** ⭐⭐⭐

- **서버 컴포넌트** (해설 페이지): 파일 시스템 접근, MDX 파싱
- **클라이언트 컴포넌트** (메인 페이지): useState로 탭 상태 관리

**면접 답변**: "서버 컴포넌트는 데이터 페칭과 무거운 연산에 사용하고, 클라이언트 컴포넌트는 useState 같은 인터랙션이 필요할 때 사용합니다. 이 프로젝트에서는 해설 페이지는 서버 컴포넌트로, 메인 페이지는 클라이언트 컴포넌트로 구현했습니다."

### 3. **MDX + KaTeX 렌더링** ⭐⭐

```tsx
<MDXRemote
  source={content}
  options={{
    mdxOptions: {
      remarkPlugins: [remarkMath], // $...$, $$...$$ 파싱
      rehypePlugins: [rehypeKatex], // HTML로 변환
    },
  }}
/>
```

**면접 답변**: "MDX는 마크다운에 JSX를 섞어 쓸 수 있는 포맷입니다. remarkMath가 수식을 파싱하고, rehypeKatex가 이를 HTML로 렌더링합니다. next-mdx-remote로 서버 사이드에서 처리했습니다."
