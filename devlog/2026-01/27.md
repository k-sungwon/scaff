## 🔑 오늘 배운 핵심 개념

### 1. CSS 변수와 Tailwind 통합 (⭐ 중요!)

**왜 CSS 변수를 쓰나요?**

- 브랜드 컬러가 바뀌면 한 곳만 수정하면 됨
- 코드에 하드코딩된 색상 값 (#FFFEF7) 대신 의미 있는 이름 (paper-light) 사용
- 다크모드 등 테마 전환이 쉬움

**Tailwind CSS v4 패턴**:

```css
/* globals.css에서 정의 */
:root {
  --paper-light: #fffef7;
}

@theme inline {
  --color-paper-50: var(--paper-light);
}
```

```tsx
// 컴포넌트에서 사용
<div className="bg-paper-50">  {/* Tailwind 유틸리티로 사용 */}
```

### 2. 상태 기반 스타일링 (⭐⭐ 매우 중요!)

이게 React UI의 핵심입니다!

```typescript
// 🎯 [직접 쳐보기] 상태에 따라 다른 스타일 적용
const [selected, setSelected] = useState("tab1");

<button
  onClick={() => setSelected("tab1")}
  className={`px-4 py-2 ${
    selected === "tab1"
      ? "bg-white text-gray-text"      // 선택됨
      : "bg-gray-100 text-gray-light"  // 선택 안됨
  }`}
>
  탭 1
</button>
```

**패턴**:

1. `useState`로 현재 상태 관리
2. 조건부 표현식으로 스타일 분기
3. 클릭 이벤트로 상태 업데이트

**면접 포인트**: "useState로 상태를 관리하고, 상태 값에 따라 조건부로 className을 적용했습니다."

### 연습 2: useState + 조건부 스타일링 ⭐⭐⭐ (가장 중요!)

이게 React UI의 90%입니다. 반드시 직접 쳐보세요!

```typescript
'use client'
import { useState } from 'react'

export default function TabExample() {
  const [activeTab, setActiveTab] = useState('tab1')

  return (
    <div>
      {/* 탭 버튼들 */}
      <div className="flex gap-2">
        <button
          onClick={() => setActiveTab('tab1')}
          className={`px-4 py-2 rounded ${
            activeTab === 'tab1'
              ? 'bg-blue-500 text-white'
              : 'bg-gray-200 text-gray-700'
          }`}
        >
          탭 1
        </button>
        <button
          onClick={() => setActiveTab('tab2')}
          className={`px-4 py-2 rounded ${
            activeTab === 'tab2'
              ? 'bg-blue-500 text-white'
              : 'bg-gray-200 text-gray-700'
          }`}
        >
          탭 2
        </button>
      </div>

      {/* 탭 내용 */}
      <div className="mt-4 p-4 bg-white rounded">
        {activeTab === 'tab1' && <div>탭 1의 내용</div>}
        {activeTab === 'tab2' && <div>탭 2의 내용</div>}
      </div>
    </div>
  )
}
```

**연습 방법**:

1. 위 코드를 직접 타이핑 (복붙 금지!)
2. 탭을 3개, 4개로 늘려보기
3. 스타일을 바꿔보기 (색상, 크기, 효과)
4. 버튼 대신 다른 UI로 바꿔보기 (카드, 리스트 등)

### 연습 3: 배열 렌더링 + 조건부 스타일 ⭐⭐

```typescript
'use client'
import { useState } from 'react'

export default function ListExample() {
  const [selectedId, setSelectedId] = useState(1)

  const items = [
    { id: 1, name: '항목 1' },
    { id: 2, name: '항목 2' },
    { id: 3, name: '항목 3' },
  ]

  return (
    <div className="space-y-2">
      {items.map(item => (
        <button
          key={item.id}
          onClick={() => setSelectedId(item.id)}
          className={`w-full p-4 text-left rounded ${
            selectedId === item.id
              ? 'bg-blue-500 text-white'
              : 'bg-gray-100 hover:bg-gray-200'
          }`}
        >
          {item.name}
        </button>
      ))}
    </div>
  )
}
```

**왜 중요한가?**:

- 메뉴, 리스트, 옵션 등 모든 선택 UI가 이 패턴
- `.map()` + `key` + 조건부 스타일 조합은 필수

## 🎤 면접 대비 질문과 답변

### Q1: CSS 변수를 사용하는 이유는?

**답변**:
"브랜드 컬러 같은 값을 한 곳에서 관리하기 위해서입니다. 예를 들어 메인 컬러가 바뀌면 CSS 변수 하나만 수정하면 전체 사이트에 적용됩니다. 또한 다크모드 같은 테마 전환도 CSS 변수를 교체하는 것만으로 가능합니다. 이 프로젝트에서는 수능 시험지 베이지, 세련된 그레이, 나비 노란색 등을 CSS 변수로 정의해서 일관된 브랜드 경험을 제공했습니다."

### Q2: useState는 언제 사용하나요?

**답변**:
"컴포넌트에서 변하는 값을 관리할 때 사용합니다. 예를 들어 이 프로젝트의 탭 UI에서는 현재 선택된 과목(selectedSubject)과 시험 종류(selectedExam)를 useState로 관리했습니다. 사용자가 탭을 클릭하면 setState로 상태를 업데이트하고, React가 자동으로 UI를 다시 렌더링해서 선택된 탭의 스타일이 변경됩니다."

### Q3: 탭 UI는 어떻게 구현하나요?

**답변**:
"세 가지 요소가 필요합니다. 첫째, useState로 현재 선택된 탭을 관리합니다. 둘째, 각 탭 버튼에 onClick으로 상태를 변경하는 핸들러를 연결합니다. 셋째, 조건부 표현식으로 선택된 탭과 아닌 탭의 스타일을 다르게 적용합니다. 이 프로젝트에서는 선택된 탭은 흰색 배경에 진한 텍스트, 선택 안 된 탭은 베이지 배경에 연한 텍스트로 구분했습니다."

### Q4: 왜 조건부 스타일링에 템플릿 리터럴을 쓰나요?

**답변**:
"기본 스타일과 조건부 스타일을 한 곳에서 관리하기 위해서입니다. 템플릿 리터럴 안에 삼항 연산자를 사용하면 조건에 따라 다른 클래스를 동적으로 적용할 수 있습니다. 예를 들어 \`className={\`px-4 py-2 ${isActive ? 'bg-blue-500' : 'bg-gray-200'}\`}\` 이렇게 작성하면 isActive 값에 따라 배경색이 자동으로 바뀝니다."
